# v0.1.11a18

Small documentation fix, found while auditing `verify-virtual-wall-write` for the same class of
"never actually run" issues already found twice elsewhere this session.

## 🔧 Fixed

- **`PrimeRobot.get_map_geojson_link()`'s own docstring was outdated**, still claiming the
  response shape/URL key name was unconfirmed. `rest_client.py`'s own docstring had already
  confirmed this (session 48, `P2MapURL$$serializer`'s own `<clinit>`: the key is `map_url`).
  Documentation-only — the underlying code was already correct.

## Also worth knowing

A full audit of `verify-virtual-wall-write` (never live-tested by anyone, unlike the other three
staged write scripts, all of which had at least one bug on first real use) did **not** turn up an
equivalent functional bug — every method it depends on (`get_map_geojson_link()`,
`download_map_bundle()`, `edit_map()`, `SetVirtualWallsV1.to_v1_command_body()`) traces back to
confirmed APK decompilation evidence, not an assumption. The real remaining risk there is "still
unconfirmed against a real device", not a known defect.
