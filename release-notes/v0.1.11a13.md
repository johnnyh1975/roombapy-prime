# v0.1.11a13

## `CurrentStateShadow` rebuilt with real, live-confirmed values

A field tester (chairstacker) shared the actual reported payload from `ro-currentstate` — not
just the key names this project had until now. Battery at 72%, robot idle and charging on its
dock. Most fields turned out to be nested objects, not flat values as originally guessed: new
`BinStatus`/`CleanMissionStatus`/`DockStatus`/`DockCapabilities`/`RuntimeStatsSummary`/
`P2MapRef` classes replace the earlier `Any`-typed placeholders.

A few concrete findings from the real data:
- Charging state lives in `clean_mission_status.phase` (observed value: `"charge"`), not a
  separate boolean.
- `dock.state`/`pw_state`/`pd_state` (301/601/701) plausibly align with `DockState`'s four
  subsystem categories found earlier via bytecode analysis — a pattern worth watching, not a
  confirmed mapping yet.
- `tank_present` is a plain boolean, confirmed genuinely distinct from any numeric tank-level
  field (none appears in this payload).

## `decode_rawmap_to_png()` — a proper library function, not just a script anymore

The confirmed rawmap-decoding logic (previously a standalone diagnostic script) is now a real
function in `models/livemap.py`. Takes raw `rawmap` bytes, returns PNG bytes, already oriented to
match the real app's own view. Pillow is an optional dependency
(`pip install "roombapy-prime[map]"`) — not required for the rest of the library.

## Fixed: a self-correction to the a12 reconnect fix

The previous release's reconnect fix relogged in on *every* reconnect attempt whenever a
`relogin` callback was configured — including an ordinary transient disconnect with a perfectly
valid token, trading a fast, simple MQTT reconnect for a full Gigya+iRobot auth round-trip
unconditionally. Narrowed: a relogin now only happens when the token is actually at/near expiry,
matching the same check `_refresh_loop()` itself already uses.

Also fixed: CI wasn't installing the optional `Pillow` dependency needed to actually exercise
`decode_rawmap_to_png()`'s own test — fixed on both sides (CI now installs it, and the test
itself skips cleanly rather than failing if Pillow genuinely isn't present).

469/469 tests green, ruff clean.
