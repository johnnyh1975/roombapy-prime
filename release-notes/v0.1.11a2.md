# v0.1.11a2

Follow-up to [v0.1.11a1](v0.1.11a1.md) — public API cleanup, documentation
accuracy, and one real bugfix found while jadestar1864 was testing
`roombapy-prime-verify-map-edit`.

## What changed

- **Real public API exports.** `roombapy_prime/__init__.py` previously
  exported nothing at all — every consumer had to reach into internal
  submodules directly (`from roombapy_prime.auth import login`, etc.),
  coupling callers to internal module layout instead of a stable contract.
  Now exports `PrimeFactory`, `PrimeRobot`, `login`, `LoginResult`,
  `RobotLoginEntry`, `AuthError`, `ShadowResponse`, with a matching
  `__all__`. This is the intended integration surface for external
  consumers — most immediately, `ha_roomba_plus`'s planned V4/Prime
  support.
- **Two stale status claims fixed.** Both `__init__.py`'s and `auth.py`'s
  module docstrings still said the login/MQTT/mission-control chain had
  never been tested against a real Prime/V4 account — true when
  originally written, no longer true since v0.1.2a0. Corrected to reflect
  the actual, current confirmation status.
- **Bugfix: `roombapy-prime-verify-map-edit` could never find a named room
  to test on, even when one existed.** `_pick_test_room()` read
  `p2map_id`/`rooms_metadata`/`name`/`room_id` via `getattr()`, but
  `robot.get_active_map_versions()` actually returns raw `list[dict]` —
  `getattr()` on a plain dict silently returns the default for every
  field, at every level, always. Confirmed via a real capture from
  jadestar1864: their account genuinely has named rooms
  (`rooms_metadata: [{"room_id": "10", "room_metadata": {"name": "Living
  Room", ...}}, ...]`) that the script reported as absent. The fix uses
  the already-existing, already-correct `parse_active_map_versions()` to
  convert the raw response before handing it to `_pick_test_room()`,
  whose own logic didn't need to change. The same bug also silently broke
  the map-bundle fallback check.
- 5 new tests total (3 for the export surface, 2 regression tests for the
  map-edit fix — one running the exact real-shaped raw dictionary through
  the actual parsing pipeline end-to-end). 350/350 tests green, ruff
  clean.

## Why

Two independent motivations landed together in this release:

1. Preparation for `ha_roomba_plus` consolidating its own login flow onto
   this library's `auth.login()` rather than maintaining a near-duplicate
   implementation in `cloud_api.py`. That consolidation needs a stable,
   documented import surface to depend on.
2. jadestar1864 ran `roombapy-prime-verify-map-edit` and shared their
   real capture — the room-finding failure it revealed was a genuine bug,
   not a data gap, and the existing unit tests didn't catch it because
   their `SimpleNamespace` test helpers used an idealized, flat shape
   that never matched what the real API actually returns.

## Known limitations

Unchanged from [v0.1.11a](v0.1.11a.md) — see that note for the full list.
This release means the live rename test in `roombapy-prime-verify-map-edit`
can now actually run for accounts with named rooms; it doesn't change
anything about how confirmed the rename/`edit_map()` call itself is (still
unverified against a live server, per the confidence table).
